# 06 · Becoming AI-Native

*Ongoing — runs in parallel from Module 2 onward*

Don't save this for the end. Fluency with AI tooling isn't a separate subject — it compounds across every other module, changing how fast you build and how deeply you can probe a concept.

## Topics

- **AI-native workflow** — using AI tools (agentic coding, LLMs as a daily work multiplier) as a genuine part of how I build, debug, and learn — not a novelty
- **Writing to understand** — turning each build into a written explanation I can reason from; the act of explaining is where understanding consolidates
- **Reflection** — noticing how working *with* AI changes the way I design, decompose problems, and catch my own mistakes

## Project (meta)

Each module's project is a real hands-on build, paired with a writeup that explains *why* it works from first principles. The writeup is for me — a forcing function for understanding — not a deliverable for anyone else. This folder collects those reflections and notes on working AI-natively.

## Done when

This is ongoing — but a good checkpoint is being able to pick up an unfamiliar AI concept, build a small thing that exercises it, and explain why it works without hand-waving. Fluency, not output count.
