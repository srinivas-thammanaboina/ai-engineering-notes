# CLAUDE.md

This repository holds my personal learning notes as I work through an AI Engineer curriculum. When helping me here, follow the guidance below.

## What this repo is

A structured set of notes, summaries, and small projects tracking my journey to becoming an AI Engineer. It is organized into six numbered modules plus a shared resources folder.

## Folder structure

- `00-resources/` — cross-module material: paper reading list, shared diagrams, cheat sheets, datasets
- `01-foundations/` — Gen AI building blocks: how LLMs work, the modern AI stack, practical constraints
- `02-rag/` — Grounding AI with RAG & context engineering
- `03-agents/` — Agentic systems: architecture, MCP & A2A, LangChain & LangGraph
- `04-finetuning/` — Finetuning fundamentals, LoRA/QLoRA, local models with Ollama
- `05-production/` — Evals, security, observability, production readiness
- `06-career/` — AI job market, portfolio building, becoming AI-native

## How to help me

- **Teach, don't just tell.** I'm learning. When I ask about a concept, explain the intuition first, then the detail. Use analogies and concrete examples.
- **Be honest and direct.** If my understanding is wrong or my code has a bug, say so plainly. Don't soften technical feedback into uselessness.
- **Keep notes consistent.** When creating or editing notes, match the existing format of the module I'm working in. Prefer clear prose with minimal formatting over walls of bullet points.
- **Tie theory to practice.** When relevant, connect a concept back to the paired paper or the module's project.
- **Don't over-engineer my projects.** These are learning exercises. Keep scope small and code readable over clever.

## Note format conventions

- One markdown file per topic, named descriptively (e.g. `chunking-strategies.md`)
- Start each note with a one-line summary of the key takeaway
- Use code blocks for any code, with the language specified
- Link to the relevant paper or source where applicable

## What I'm optimizing for

Genuine understanding I can apply and explain in an interview — not just completed checkboxes. Push me toward building and writing things up, not just reading.
